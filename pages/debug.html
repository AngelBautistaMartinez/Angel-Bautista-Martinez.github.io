Anti-Debug

1. Checking Debugger in Parent Process

Malware can check what its parent function is. If it's running a parent process that's a debugged process, it indicates a debugger. The child process could invoke DebugActiveProcess with its parent process as a parameter. If the parent process (the original malware process) is already being debugged, this function will throw an exception, indicating to the malware that there's a debugger attached to it.

C/C++ Code 
#include <windows.h>
#include <tlhelp32.h>
#include <stdio.h>

int CheckForDebug() {
    //get this programs process ID
    DWORD currentPid = GetCurrentProcessId();   
    //set up for parent process ID
    DWORD parentPid = 0;                        

    printf("Current PID: %lu\n", currentPid);
    //get snapshot of all processes
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPPROCESS, 0);

    if (hSnapshot == INVALID_HANDLE_VALUE) {
        printf("Snapshot failed, error %lu\n", GetLastError());
        return -1;
    }
    //set up process structure
    PROCESSENTRY32 pe;
    pe.dwSize = sizeof(PROCESSENTRY32);

    //loop through processes
    if (Process32First(hSnapshot, &pe)) {
        do {
            //if we find the parent process 
            if (pe.th32ProcessID == currentPid) {
                parentPid = pe.th32ParentProcessID;   
                break;                               
            }
        } while (Process32Next(hSnapshot, &pe));
    }
    //close handle
    CloseHandle(hSnapshot);

    if (parentPid == 0) {
        printf("Could not find parent PID.\n");
        return -1;
    }

    printf("Parent PID: %lu\n", parentPid);

    //if the DebugActiveProcess function does not work
    if (!DebugActiveProcess(parentPid)) {
        return 1;
    } else {
        //if we can act as a debugger to our parent process
        printf("Successfully attached to parent.\n");
        DebugActiveProcessStop(parentPid); 
    }

    return 0;
}


int main(){
    if(CheckForDebug() == 0){
        printf("Parent Process not being debugged.\n");
    }
    else if(CheckForDebug() == 1){
        printf("!Process being Debugged!\n");
        
    }
    return 0;
}

2. Calling DebugBreak to see Exception

If malware executes the instruction int 3 or DebugBreak() and it's not being debugged, an EXCEPTION_BREAKPOINT will be raised and the exception handler will be invoked. Of course, the opposite is also true. If this EXCEPTION_BREAKPOINT doesn't invoke the exception, that means the debugger probably caught the Exception.

Note: I didn't use __asm(int3) because of my system component, so I used DebugBreak() and used my own handler to see what exception we got.

C/C++ Code 
#include <stdio.h>
#include <windows.h>

//make a dummy function to pose as a real funtion the malware would have
void dummyFunction1(){
    printf("I am dummy function 1\n");
}

//we determine what happens when theres an exception
//argument is a exception structure we will use to get info about the exception
LONG WINAPI ownHandler(EXCEPTION_POINTERS *ExceptionInfo){
    //we go from info to record to code in the struct to see if we have a BREAKPOINT(aka no debugger caught the exception)
    if(ExceptionInfo->ExceptionRecord->ExceptionCode == EXCEPTION_BREAKPOINT){
        printf("No Debugger Program caught DebugBreak Exception\n");
        //skips debug instruction we called
        ExceptionInfo->ContextRecord->Rip++;

        return EXCEPTION_CONTINUE_EXECUTION;
    }
    return EXCEPTION_CONTINUE_SEARCH;
}

int main(){
    //lets us implement our own handler
    //1 = our handle gets called first to be able to see exceptions
    AddVectoredExceptionHandler(1, ownHandler);
    dummyFunction1();
    DebugBreak();
    return 0;   
}
